PAdES Core

Motor experimental para gerar e validar assinaturas digitais PDF compativeis com PAdES, com foco em CMS interno, atualizacao incremental e API publica simples para aplicacoes PHP.

Licenca MIT Pacote nihillabs/pades-core PHP 8.4+ Status Experimental

Assinatura PDF-first

Cria assinaturas PAdES diretamente em PDFs, preservando os bytes anteriores com incremental update.

CMS interno

Gera SignedData detached com SigningCertificateV2, ESSCertIDv2, certificados X.509 e compatibilidade OpenSSL.

API de aplicacao

Expoe importacao de PFX, assinatura com certificado salvo, `Pades::sign` e `PadesSigner` para controle fino.

Quick Start

Instale o pacote, importe o PFX/P12 enviado pelo usuario, salve o conteudo do certificado sem a senha e solicite a senha novamente quando for assinar.

1

Instale

Adicione a biblioteca pelo Composer.

2

Importe

Valide o PFX/P12 e extraia metadados com a senha digitada no momento.

3

Assine

Use o certificado salvo e a senha digitada novamente para gerar o PDF assinado.

composer require nihillabs/pades-core
<?php

require __DIR__ . '/vendor/autoload.php';

use NihilLabs\Pades\Certificate\PfxCertificateImporter;
use NihilLabs\Pades\PadesSigner;
use NihilLabs\Pades\Signing\PfxSignatureCredential;

$pfxContents = file_get_contents($_FILES['certificate']['tmp_name']);

if ($pfxContents === false) {
    throw new RuntimeException('Nao foi possivel ler o certificado enviado.');
}

$metadata = (new PfxCertificateImporter())->inspectContents(
    contents: $pfxContents,
    password: $passwordTypedOnImport
);

// Salve $pfxContents e os metadados necessarios.
// Nao salve a senha.

(new PadesSigner())->sign(
    inputPdf: __DIR__ . '/documento.pdf',
    outputPdf: __DIR__ . '/documento-assinado.pdf',
    credential: PfxSignatureCredential::fromContents(
        contents: $storedPfxContents,
        password: $passwordTypedWhenSigning
    )
);

Instalacao

O pacote e uma biblioteca Composer com autoload PSR-4 no namespace `NihilLabs\Pades\`.

composer require nihillabs/pades-core

Requisitos

  • PHP 8.4 ou superior.
  • Extensao OpenSSL habilitada.
  • Composer configurado no projeto consumidor.

Setup

Senhas e certificados reais devem ficar fora do repositorio e fora de diretorios publicos. A aplicacao deve receber ou localizar o PDF original, chamar a biblioteca e entregar o arquivo assinado ao usuario.

A biblioteca nao exige `.env`. Variaveis de ambiente sao apenas uma opcao para desenvolvimento local; em um sistema com certificado importado pelo usuario, solicite a senha no momento da importacao e novamente no momento da assinatura.

export PFX_PASSWORD='sua-senha-do-pfx'
export ICP_PFX_PASSWORD='sua-senha-do-certificado-icp-local'
$passwordTypedWhenSigning = $request->input('certificate_password');

if (! is_string($passwordTypedWhenSigning) || $passwordTypedWhenSigning === '') {
    throw new RuntimeException('Senha do certificado nao informada.');
}

Para producao, trate a senha como dado efemero: use somente durante a chamada, nao persista, nao registre em logs e nao coloque em query string.

Importacao de PFX

Quando o usuario envia o certificado, use a senha apenas para validar o PFX/P12 e extrair metadados. Salve o conteudo do certificado em banco ou storage seguro e nao salve a senha.

use NihilLabs\Pades\Certificate\PfxCertificateImporter;

$pfxContents = file_get_contents($_FILES['certificate']['tmp_name']);

if ($pfxContents === false) {
    throw new RuntimeException('Nao foi possivel ler o certificado enviado.');
}

$metadata = (new PfxCertificateImporter())->inspectContents(
    contents: $pfxContents,
    password: $passwordTypedOnImport
);

// Salve $pfxContents em banco/storage seguro.
// Salve metadados como commonName, serialNumberHex, validFrom e validTo.
// Nao salve a senha do certificado.

Metadados retornados

Campo Uso comum
`commonName` Nome exibido para o usuario identificar o certificado.
`serialNumberHex` Identificador tecnico para auditoria e conferencia.
`subject` / `issuer` Dados do titular e da autoridade emissora.
`validFrom` / `validTo` Janela de validade para avisos antes da assinatura.
`certificatePem` / `certificateChainPem` Material publico do certificado e cadeia.

Certificado salvo

No momento da assinatura, solicite a senha novamente e crie a credencial a partir dos bytes armazenados. O PDF assinado continua sendo gravado no caminho `outputPdf`.

use NihilLabs\Pades\PadesSigner;
use NihilLabs\Pades\Signing\PfxSignatureCredential;

$credential = PfxSignatureCredential::fromContents(
    contents: $storedPfxContents,
    password: $passwordTypedWhenSigning
);

(new PadesSigner())->sign(
    inputPdf: '/tmp/documento.pdf',
    outputPdf: '/tmp/documento-assinado.pdf',
    credential: $credential
);

Atalho estatico

use NihilLabs\Pades\Pades;

Pades::signWithPfxContents(
    inputPdf: '/tmp/documento.pdf',
    outputPdf: '/tmp/documento-assinado.pdf',
    certificateContents: $storedPfxContents,
    certificatePassword: $passwordTypedWhenSigning
);

API simples: `Pades::sign`

Use quando o certificado PFX/P12 estiver disponivel como arquivo. Para certificados salvos em banco/storage, prefira `Pades::signWithPfxContents` ou `PfxSignatureCredential::fromContents`.

use NihilLabs\Pades\Pades;
use NihilLabs\Pades\PadesSignatureOptions;

Pades::sign(
    inputPdf: '/tmp/documento.pdf',
    outputPdf: '/tmp/documento-assinado.pdf',
    certificatePath: '/secure/certificate.pfx',
    certificatePassword: $password,
    options: new PadesSignatureOptions(
        visibleSignature: true,
        appendSignaturePage: true,
        signatureName: 'Nome do assinante',
        signatureReason: 'Assinatura digital',
        signatureLocation: 'Sistema interno',
        signatureContactInfo: 'assinante@example.com'
    )
);
Parametro Uso
`inputPdf` Caminho absoluto ou relativo do PDF original.
`outputPdf` Caminho onde o PDF assinado sera escrito.
`certificatePath` Caminho do arquivo PFX/P12.
`certificatePassword` Senha do PFX/P12, recebida de forma efemera no momento da assinatura.

API recomendada: `PadesSigner`

Use quando a aplicacao precisa controlar credencial, TSA, assinatura visivel, campo existente, algoritmo, locks ou provider de assinatura.

use NihilLabs\Pades\PadesSignatureOptions;
use NihilLabs\Pades\PadesSigner;
use NihilLabs\Pades\Signing\PfxSignatureCredential;

$credential = new PfxSignatureCredential(
    pathOrCertificate: '/secure/certificate.pfx',
    password: $password
);

(new PadesSigner())->sign(
    inputPdf: '/tmp/documento.pdf',
    outputPdf: '/tmp/documento-assinado.pdf',
    credential: $credential,
    options: new PadesSignatureOptions(
        visibleSignature: true,
        appendSignaturePage: true,
        signatureName: 'Nome do assinante',
        signatureReason: 'Assinatura digital'
    )
);

Credenciais

A biblioteca suporta PFX/P12, PEM e providers desacoplados para HSM, PKCS#11, smartcard, cloud KMS, remote signing ou callback externo.

PFX/P12 em arquivo

use NihilLabs\Pades\Signing\PfxSignatureCredential;

$credential = new PfxSignatureCredential(
    pathOrCertificate: '/secure/certificate.pfx',
    password: $password
);

PFX/P12 em bytes

use NihilLabs\Pades\Signing\PfxSignatureCredential;

$credential = PfxSignatureCredential::fromContents(
    contents: $storedPfxContents,
    password: $passwordTypedWhenSigning
);

PEM

use NihilLabs\Pades\Signing\PemSignatureCredential;

$credential = PemSignatureCredential::fromFiles(
    certificatePath: '/secure/certificate.pem',
    privateKeyPath: '/secure/private-key.pem',
    privateKeyPassword: $passwordOrNull
);

Opcoes

`PadesSignatureOptions` concentra o comportamento da assinatura: TSA, assinatura visivel, pagina final, campo existente, locks, tipo de assinatura e algoritmos.

new PadesSignatureOptions(
    timestampProvider: null,
    visibleSignature: false,
    signatureRect: [48, 48, 547, 96],
    signatureFlags: 132,
    signatureName: 'PAdES Core',
    signatureReason: 'Document signed digitally',
    signatureLocation: null,
    signatureContactInfo: null,
    signatureFieldName: null,
    signatureType: PadesSignatureOptions::SIGNATURE_TYPE_APPROVAL,
    certificationPermission: 2,
    lockedFieldNames: [],
    fieldLockAction: PadesSignatureOptions::FIELD_LOCK_INCLUDE,
    hashAlgorithm: 'sha256',
    signatureAlgorithm: 'rsa',
    minimumHashAlgorithm: 'sha256',
    includeSigningTime: false,
    appendSignaturePage: false,
    signaturePageMediaBox: [0, 0, 595, 842],
    signaturePageRect: [48, 120, 547, 700]
);

Campo existente

Use `signatureFieldName` para assinar um campo `/FT /Sig` vazio ja existente no PDF. A biblioteca suporta nome totalmente qualificado em AcroForm hierarquico e respeita Seed Value Dictionary e `/Lock` quando presentes.

new PadesSignatureOptions(
    signatureFieldName: 'Section.Approval',
    signatureName: 'Nome do assinante',
    signatureReason: 'Aprovado'
);

Se o campo nao existir ou ja estiver assinado, a biblioteca tenta criar um novo campo quando possivel.

Certificacao e locks

Configure assinatura de certificacao e travamento de campos quando o PDF precisar restringir alteracoes posteriores.

new PadesSignatureOptions(
    signatureType: PadesSignatureOptions::SIGNATURE_TYPE_CERTIFICATION,
    certificationPermission: 2,
    lockedFieldNames: ['Amount', 'Approval'],
    fieldLockAction: PadesSignatureOptions::FIELD_LOCK_INCLUDE
);
  • Adiciona DocMDP quando `signatureType` e `certification`.
  • Adiciona FieldMDP quando `lockedFieldNames` ou lock do campo exigirem.

Timestamp TSA

Informe um provider RFC 3161 quando quiser gerar assinatura com timestamp. Sem TSA, o fluxo e PAdES-B-B; com TSA, o fluxo esperado e PAdES-B-T.

use NihilLabs\Pades\Crypto\Timestamp\HttpTimestampClient;
use NihilLabs\Pades\PadesSignatureOptions;

$options = new PadesSignatureOptions(
    timestampProvider: new HttpTimestampClient('https://freetsa.org/tsr')
);

Para validacao oficial, use uma TSA confiavel para o validador alvo. FreeTSA serve para teste tecnico, mas pode ficar indeterminada em validadores oficiais.

Perfil Status documentado Observacao

Validacao

Os testes automatizados cobrem assinatura PDF real, CMS interno, ByteRange, aparencia visivel, fonte nao embutida e validacao estrutural. Validadores como Adobe, DSS e leitores nativos ainda entram em fluxos manuais ou de interoperabilidade.

vendor/bin/phpunit
vendor/bin/phpunit --filter RealPdfSignerTest
vendor/bin/phpunit --filter 'PdfSignatureAppearanceTest|PdfFontEmbeddingInspectorTest|RealPdfSignerTest'

Leitores manuais

  • Adobe Acrobat e Adobe Reader.
  • Google Chrome e Mozilla Firefox.
  • Preview do Windows ou macOS quando aplicavel.

Seguranca operacional

  • Nao persista senha de certificado em banco, `.env`, log ou arquivo temporario.
  • Use a senha apenas durante importacao ou assinatura e descarte depois da chamada.
  • Nao grave senha no banco em texto puro.
  • Proteja o PFX fora de diretorios publicos.
  • Use arquivos temporarios fora de `public/`.
  • Valide tamanho e tipo do PDF antes de assinar.
  • Nao versione certificados ICP reais, chaves privadas reais, PDFs privados ou saidas de teste.

Escopo atual

O fluxo principal esperado recebe um PDF nao criptografado, aplica assinatura incremental, gera CMS/CAdES detached internamente, preenche `/ByteRange` e `/Contents`, inclui `SigningCertificateV2` e cria campo invisivel ou visivel.

Suportado no fluxo principal

  • PDF nao criptografado.
  • Assinatura invisivel ou visivel.
  • Pagina final dedicada quando configurada.
  • Suporte a PFX/P12 e PEM.
  • Validacao estrutural basica de ByteRange.

Ainda experimental

  • PAdES-B-T, PAdES-LT e PAdES-LTA.
  • DSS, VRI, OCSP, CRL e trust store completo.
  • PDFs criptografados ou com formularios complexos.
  • Validacao completa de CMS/CAdES sem ferramenta externa.
